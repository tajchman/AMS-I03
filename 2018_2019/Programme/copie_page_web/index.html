Prérequis

Il est conseillé aux étudiants qui souhaitent suivre ce cours, d'avoir déjà utilisé des machines de calcul sous linux et les outils standards (shell, editeurs, etc) disponibles dans cette plateforme. La connaissance d'un langage de programmation utilisé dans le domaine scientifique, par exemple C, C++ ou fortran, est indispensable pour effectuer les travaux pratiques demandés. Enfin, on supposera que les étudiants ont de bonnes notions en programmation parallèle avec MPI, obtenues par exemple en suivant Cours AMS301 - Calcul Scientifique Parallèle.

Programme du cours

Le programme ci-dessus est indicatif, il est susceptible de légères adaptations.
  • Rappels sur les architectures des machines et différents types de programmation parallèle (1/2 séance)
  • Optimisation de la programmation séquentielle (1/2 séance)
    • Mémoire cache des machines.
    • Notions de localités spatiale et temporelle.
    • Utiliser les possibilités du processeur.
    • Influence sur l'efficacité des algorithmes.
    • Exemples.
    • Enoncé du TP 1 et fichiers de code source associés
  • Programmation multi-threads (2 séances).
    • Différents outils disponibles.
    • Exemples avec OpenMP et std::threads.
    • Partage ou non des variables.
    • Différentes techniques de programmation avec OpenMP (grain fin, grain grossier, programmation par tâches).
    • Enoncé du TP 2 et fichiers de code source associés
    • Enoncé du TP 3 et fichiers de code source associés
  • Programmation hybride MPI - OpenMP (2 séances).
    • Intérêt de ce type d'approche.
    • Comparaison de différentes techniques MPI-OpenMP sur un code de calcul.
    • Exposé
    • Enoncé du TP 4 et fichiers de code source associés
    • Enoncé du TP 5 et fichiers de code source associés
  • Programmation des GPU (Graphics Processing Unit) (4 séances).
    • Caractéristiques principales.
    • Différents outils disponibles.
    • Exemples avec le language Cuda.
    • Points importants pour une utilisation efficace du GPU.
  • Examen (1 séance).
